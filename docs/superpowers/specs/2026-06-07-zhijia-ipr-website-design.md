## 目标

基于本地知识库 `C:\Users\57355\Documents\Obsidian Vault\知识产权` 搭建一个面向客户的高端知识产权服务官网。网站优先承担品牌展示、服务解释、客户咨询转化；内部知识库能力作为轻量入口保留，不作为第一视觉重点。

第一版不做复杂后台、CRM、在线支付、真实 AI 问答或账号权限系统。先把公开官网、内容结构化、服务检索和表单+电话转化做好。

## 已确认方向

- 网站定位：客户官网优先，不是纯文档站。
- 视觉风格：参考 `https://www.zhijiaipr.com/`，但升级为更高端的 B2B 服务品牌风格。
- 视觉关键词：深蓝、香槟金、少量科技蓝、大片留白、真实图片、克制动效、专业可信。
- 主转化方式：表单 + 电话并列突出。
- 内部入口：保留轻量入口，放在导航或页脚，不打扰客户访问。
- 国家/地区：国家名称区域使用国旗图标，正式实现建议使用稳定图标库，不依赖 emoji。

## 技术路线

推荐栈：

- Next.js App Router
- TypeScript
- Tailwind CSS
- shadcn/ui
- 结构化 JSON/MDX 内容层
- Fumadocs 作为内部知识入口和搜索体验的可选增强，不控制公开官网页面布局
- 本地静态搜索，后续可升级 Algolia / Meilisearch
- 表单第一版先写入本地提交记录或调用轻量 API，后续接入邮箱、飞书、企业微信或 CRM

选择理由：

- Next.js 适合官网、SEO、表单、后续后台和 Vercel 部署。
- Tailwind + shadcn/ui 适合构建高端、克制、可维护的 B2B UI。
- 公开官网页面使用自定义设计组件；内部知识入口可复用 Fumadocs 的目录和搜索能力，但不把公开页做成传统程序员文档风格。
- 第一版以静态内容生成为主，降低复杂度和维护成本。

## 页面视觉规范

### 品牌气质

目标不是廉价营销页，而是高端专业服务官网。

关键词：

- 高端
- 稳妥
- 国际化
- 顾问式服务
- 专业可信

### 色彩

主色：

- 深蓝：用于首屏、页脚、深色流程区。
- 科技蓝：用于链接、图标、强调信息。
- 香槟金：用于高端感、关键按钮、步骤点和细线。
- 白色/象牙白：用于大面积留白和内容承载。

避免：

- 大面积单调蓝紫渐变
- 过度拥挤的卡片堆叠
- 花哨廉价动效
- 信息密度过高的首屏

### 图片资产

第一版建议使用三类图片：

- 高端首屏图：行政办公、全球地图、证书、深蓝金色氛围。
- 顾问服务图：顾问会议、材料核对、企业出海咨询。
- 证书/全球保护图：证书、国旗、全球网络、品牌保护。

正式实现时图片应放入项目资产目录，使用 Next.js 图片优化，不使用草图阶段的内嵌 base64。

### 国旗图标

Demo 使用 emoji 表达方向。正式实现建议使用 `flag-icons` 或同类 SVG/CSS 图标库，保证：

- 跨平台显示一致
- 可控尺寸和圆形裁切
- 不依赖系统 emoji 字体
- 易于按国家代码映射

马德里注册不是国家，不使用国旗，使用全球/组织类图标。

### 动效层级

动效保持克制：

- 首屏品牌轨道慢速旋转
- 卡片 hover 轻微上浮
- 流程连接线可做轻微流光
- 页面进入视口时淡入/上移
- 表单按钮 hover 轻微阴影变化

不做：

- 大量弹跳动画
- 眩光过度扫屏
- 自动滚动干扰阅读
- 页面元素频繁抖动

## 内容转换方案

不直接把 Obsidian Markdown 原样发布。

推荐建立转换脚本：

1. 读取源 Markdown。
2. 解析二级标题：`## 国家/地区 - 业务`。
3. 解析三级标题：`需要材料`、`材料说明`、`申请材料补充`、`提醒/话术`、`售后/流程` 等。
4. 生成结构化内容：
   - `serviceDomain`
   - `country`
   - `countryCode`
   - `serviceType`
   - `publicSummary`
   - `materials`
   - `process`
   - `riskNotes`
   - `internalTalkTracks`
   - `sourceFile`
   - `sourceHeading`
5. 网站页面只读取结构化内容，不直接依赖原始长 Markdown 页面。

公开内容与内部内容分离：

- 公开：材料、流程、适用场景、风险提示。
- 内部：话术、售后提醒、敏感口径。
